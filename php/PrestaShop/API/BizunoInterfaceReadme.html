Created 2015-04-17
Last Update 2015-04-17
Tested with Bizuno Release 1.0 and PrestaShop Release 1.5.6
Developed by PhreeSoft, LLC. (www.PhreeSoft.com)

Requirements:

  • PhreeBooks R3.6 RC3 or greater

  • Bizuno 1.0 or greater

  • PrestaShop 1.5.6 or greater (with vqmod installed)

NOTE: The PHP cURL functions are required at both sides for the XML transactions to work properly.

NOTE: Translation for English is included.

New Installation - Bizuno Side

  1. Go to My Business->Settings, From there go to the Interfaces tab and click install PrestaShop.

  2. Go to My Business->Roles, Create a new user (and password) and set all permissions to None except the PrestaShop Interface (set to Full).
    This user name will be used by PrestaShop to insert data into PhreeBooks.

  3. Log out and back in to reload the security settings.

SETUP - Bizuno Side

  1. Edit the PrestaShop Properties and enter the information.
    PrestaShop path: full url to path where the upload file is located. It is mostly set be default, the domain and path to the root admin folder needs to be set.
    PrestaShop username/password: An PrestaShop admin user name and password to allow verification of the interface when uploading data. This should be a unique user with permissions only for the Bizuno/PhreeBooks interface.

  2. The install script will create a new tab in your inventory maintenance page along with new fields. These new fields are required to properly sync your current products with the products in the PrestaShop store. Check Inventory -> Edit/Maintain, Open a product and check the new e-Store tab is correctly showing the extra fields.

New Installation - PrestaShop Side

In the PrestaShop_Bizuno_API/ folder are two folders, modules and admin (or whatever your admin folder has been renamed to). All files are PhreeBooks specific and will not overwrite core PrestaShop files during this step.

  1. Admin Files: Copy the contents of the admin folder to your PrestaShop admin folder (which should have been renamed from the default installation name of “admin” as per PrestaShop security recommendations).

  1. modules Files: Copy the contents of the vqmod folder to the vqmod folder.

    SETUP – PrestaShop Side

    1. Go to System → Users → User Groups and set permissions for the groups to access the module. Set both Access and Modify permissions for the Top Administrator. The title is PhreeBooks/orders.

    2. Go to Modules (Export) → Bizuno Interface → Order Downloads and click install.

    3. Click edit and set the following:

      • Enter the Bizuno/PhreeBooks user name created for downloading orders.

      • Enter the Bizuno/PhreeBooks password for the user.

      • For the PhreeBooks URL enter: http://path_to_PhreeBooks_root (or blank if bizuno)

      • If using multi_store, enter a store id on Download Company. This can be used for reporting activity through different sales channels. This field is not required and may be left blank.

      • Enter an optional prefix (e.g. OC) to use for creating the sales order number. Note PhreeBooks does not allow for duplicate Sales Order numbers. Adding a prefix will prevent conflict between downloaded orders and orders generated within PhreeBooks.

      • Select the status you want to set the order to after download. Typically I use processing to differentiate from Pending when the order is placed by the customer.

    • Categories in PhreeBooks must exist in PrestaShop and match identically. This is used to properly determine where to place the item in the PrestaShop catalog.

    USAGE - PhreeBooks Side

    Product Related

    1. Inventory → Maintain
      Once the added fields have been entered into Bizuno/PhreeBooks, the catalog checkbox is tested for each inventory item and if selected, an upload icon will appear next to the inventory item in the action area of the main inventory screen. Click the button to upload the product.

    2. Tools → PrestaShop Interface → Bulk Upload
      This action will upload all inventory items that have the catalog checkbox set in the inventory database. Images can be excluded to speed the upload process.

    3. Tools → PrestaShop Interface → Synchronize Products
      This feature will compare the selected inventory items from the Bizuno/PhreeBooks inventory database file and verify they match with the products listed on your PrestaShop e-store.
      Bizuno/PhreeBooks will not delete the items on the PrestaShop store but will identify items that are there but are not flagged to be there. They must be removed manually from PrestaShop.

    Order Related

    1. Tools → PrestaShop Interface → Confirm Shipments
      This feature will change the status for all orders shipped on the selected date. Data is pulled from the shipping_log table and will include the carrier and tracking number.
      The order status will be changed to the value set during the PrestaShop Administration setup.

    USAGE - PrestaShop Side

    Product Upload

    No action (data is received)

    Order Download

    1. Individual Order Download - Once an order has been placed, select Sales → Orders from the admin screen.
      An [download] link will appear in the action area next to each order to download the order.
      Click the link to download the order to PhreeBooks. If successful, the download link will no longer appear. Repeat for all other orders.
